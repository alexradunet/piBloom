# Pi 0.60.0 Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade pi from 0.57.1 to 0.60.0, replacing the `llm-agents-nix` flake dependency with a direct `buildNpmPackage` derivation, and integrate `createLocalBashOperations()` into `bloom-audit` for automatic bash command audit logging.

**Architecture:** A new `core/os/pkgs/pi/default.nix` derivation uses `buildNpmPackage` to build pi from the npm registry, mirroring how `bloom-app` is packaged. The `llm-agents-nix` flake input is removed from `flake.nix` along with its ISO embedding. The `bloom-audit` extension is extended to wrap pi's bash backend and log every command invocation and result.

**Tech Stack:** Nix (`buildNpmPackage`, `fetchurl`), TypeScript/Vitest (bloom-audit extension), NixOS modules.

**Spec:** `docs/superpowers/specs/2026-03-18-pi-0.60.0-upgrade-design.md`

- [ ] **Step 3.4: Update `isAuditEntry` guard in `actions.ts`**

The guard currently only accepts `tool_call` and `tool_result`. Extend it to also accept the two new event types so bash entries are not silently dropped by `readEntries`:

```ts
function isAuditEntry(value: unknown): value is AuditEntry {
  if (!value || typeof value !== "object") return false;
  const entry = value as Record<string, unknown>;
  const validEvents = new Set(["tool_call", "tool_result", "bash_invoke", "bash_result"]);
  return (
    typeof entry.ts === "string" &&
    typeof entry.event === "string" &&
    validEvents.has(entry.event) &&
    typeof entry.tool === "string" &&
    typeof entry.toolCallId === "string"
  );
}
```

Also update `formatEntries` to handle bash event display — bash results use `exitCode` not `isError`:

```ts
export function formatEntries(entries: AuditEntry[], includeInputs: boolean): string {
  const lines: string[] = [];
  for (const e of entries) {
    let status: string;
    if (e.event === "tool_result") {
      status = e.isError ? "error" : "ok";
    } else if (e.event === "bash_result") {
      status = e.exitCode === 0 ? "ok" : "error";
    } else {
      status = "call";
    }
    lines.push(`- ${e.ts} ${e.tool} [${status}]`);
    if (includeInputs && (e.event === "tool_call" || e.event === "bash_invoke")) {
      const input = summarizeInput(e.input);
      if (input) lines.push(`  input: ${input}`);
    }
  }
  return truncate(lines.join("\n"));
}
```
