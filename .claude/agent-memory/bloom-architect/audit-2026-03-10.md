### lib/ Purity (functions do I/O)
Pure functions that read/write disk (acceptable for lib/ since functions take explicit path args):
- services-catalog.ts: readFileSync, existsSync (I/O in function bodies, not module level)
- services-install.ts: heavy I/O (writeFileSync, mkdirSync, rmSync, cp, podman)
- services-manifest.ts: readFileSync, writeFileSync
- services-validation.ts: readFileSync, existsSync
These are NOT "pure" per ARCHITECTURE.md rule 1: "No side effects, no I/O at module level"
The rule says "Functions take inputs, return outputs" but these functions DO I/O.
