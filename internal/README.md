# Internal Documentation

This tree holds non-public project material that should not be published through the VitePress site.

Use it for:

- maintainer-only docs about keeping the docs site clean
- workflow notes that are useful internally but not part of the public product docs
- design specs and planning artifacts

Keep `docs/` reserved for current public documentation only.

Suggested layout:

- `internal/maintainers/` — maintainer-only process notes
- `internal/workflows/` — internal operating workflows
- `internal/specs/` — design and planning artifacts
